OPNsense Edge Firewall

Bare-metal firewall and routing project focused on the fundamentals that matter: traffic flow, DNS, DHCP, NAT, secure remote access, IDS concepts, and careful documentation of what is exposed and what stays internal.

OPNsense Firewall Rules DNS / DHCP NAT Review Suricata Concepts

What I Built

  • Deployed OPNsense on dedicated hardware as the edge router and firewall.
  • Configured LAN routing, DHCP, DNS, and service access patterns.
  • Used Cloudflare DNS-01 and wildcard certificate workflows for services.
  • Reviewed WAN NAT forwards and documented which services are intentionally reachable.
  • Mapped firewall behavior into public-safe documentation without exposing private config.

Skills Demonstrated

  • Network segmentation and routing fundamentals
  • Firewall rule review and change tracking
  • DNS resolver and filtering concepts
  • Public/private service boundary thinking
  • Security documentation and operational judgment
DNS

Filtering and Resolution

Documented the move toward DNS filtering and upstream encrypted resolver concepts, with notes on enforcement that still need live verification.

Ingress

Controlled Exposure

Separated public-facing service access from private management access and marked uncertain paths for verification instead of guessing.

Security

IDS Awareness

Included Suricata/IDS concepts as part of the edge design and kept the public portfolio focused on architecture rather than sensitive rules.

What This Shows Employers

I can think like a technician responsible for a live edge device: understand traffic direction, avoid exposing management services casually, document firewall state, and keep sensitive configuration out of public materials.
