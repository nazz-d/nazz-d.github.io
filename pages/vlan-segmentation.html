Enterprise VLAN Segmentation

Redesigned a flat homelab network from a single 192.168.1.0/24 broadcast domain to a fully segmented 10-VLAN production architecture — each VLAN with a defined security posture, explicit OPNsense firewall rules, and zero unnecessary cross-VLAN exposure. 30+ services migrated, 80+ firewall rules enforced.

OPNsense 802.1Q VLANs Cisco SG300-20 Netgear GS728TP Firewall Design Default-Deny DNS Migration Proxmox

What I Built

  • 10-VLAN Design: Segmented the entire lab across 10 VLANs on a 10.226.X.0/24 scheme — management, game servers, DMZ, services, storage, servers, IoT, testing, guest, and OOB management each isolated at the firewall.
  • OPNsense Default-Deny: Enforced 80+ explicit allow rules across all VLAN interfaces — if there is no rule, traffic is blocked. Every cross-VLAN path that needs to work is documented as a firewall rule.
  • Switch Hardware Swap: Replaced legacy Cisco Catalyst 3750V2 (100 Mbps) switches mid-migration with Cisco SG300-20 core + Netgear GS728TP PoE+ access. Re-trunked all VLANs from scratch.
  • Proxmox Quorum Recovery: Restored 3/3 corosync quorum after the switch swap left VLAN 10 absent from Proxmox node ports — identified via LLDP, fixed via SG300 trunk config.
  • Bulk Service Migration: Moved 30+ VMs and LXC containers to correct VLANs, updated 25 DNS host overrides via OPNsense API, reconfigured 18 NPMplus reverse proxy backends, and updated 7 NAT port-forward rules.
  • DMZ Architecture: cloudflared and NPMplus isolated in VLAN 7. NPMplus proxies to approved backend IPs only — no DMZ host can initiate connections to management or storage.

Security Design Decisions

  • VLAN 10 — Management: Admin PC, Proxmox nodes, switches, and OPNsense management interface. The only VLAN allowed to initiate connections everywhere else. No service VLAN can reach VLAN 10 without an explicit rule.
  • VLAN 7 — DMZ: cloudflared (Cloudflare Tunnel) and NPMplus (reverse proxy) are the only hosts. cloudflared is permitted to reach NPMplus only. NPMplus is permitted to reach approved backend IPs on specific ports. Nothing in DMZ initiates to management.
  • VLAN 20 — Storage: Synology NAS, QNAP, and Thecus PBS cannot initiate outbound internet connections. VLAN 10 and Proxmox nodes are the only sources permitted to reach NFS/SMB ports.
  • VLAN 8 / VLAN 5 — Guest & IoT: Hard RFC1918 block — these VLANs get internet access and nothing else. Guest uses OPNsense captive portal. IoT devices cannot reach any infrastructure VLAN.
  • Wazuh log collection: Every VLAN has per-interface rules allowing outbound to Wazuh (VLAN 3) on ports 1514, 1515, and 514. Security monitoring without general cross-VLAN access.
Addressing

Structured IP Scheme

Every VLAN follows 10.226.X.0/24 where X is the VLAN ID. Within each subnet: .1 is the OPNsense gateway, .2–.49 are static infrastructure IPs, .50–.99 are reserved for services, and .100–.200 is the DHCP pool. Predictable and easy to firewall-rule by subnet.

Enforcement

OPNsense Firewall Policy

OPNsense applies rules per VLAN interface. Default-deny means any unlisted traffic is silently dropped. Each new service requires a conscious decision: which VLAN source, which destination IP and port, and why. No implicit trust between any two VLANs — not even adjacent ones.

Public Ingress

DMZ-Only Internet Exposure

All public-facing services reach the internet exclusively through Cloudflare Tunnel in VLAN 7. No management ports, no Proxmox UI, no direct SSH — none of it is reachable from the WAN. The only path from the internet into the lab is cloudflared → NPMplus → approved backend IP.

What This Shows Employers

This project demonstrates full-cycle network architecture work — not just plugging in a switch, but designing a security model, implementing it across hardware and software layers, recovering from failures mid-migration, and validating every traffic path against an explicit policy. The result is a production network where every device is isolated to exactly the access it needs, public services are completely separated from management, and every cross-VLAN flow is documented as a deliberate firewall rule. That's the kind of thinking that applies directly to enterprise network administration, NOC work, and security operations.
